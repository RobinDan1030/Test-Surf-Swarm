# Boid 群体行为设计文档

## 一、老大鸟（Leader Boid）

### 1.1 角色定义
- **唯一性**: 场景中存在且仅存在一个老大鸟
- **控制方式**: 由环境和算法控制（待定）
- **能力**: 具备环境感知和避障能力

### 1.2 核心职责
- 向下级小鸟发送**运动向量**（唯一指令类型）
- 所有 Boid 速度大小相同，向量本身即包含避障信息

### 1.3 引力系统（仅老大鸟）
- **引力点**: 由玩家操控的目标点，用于吸引老大鸟前往
- 老大鸟根据引力点位置和环境算法决定行为

---

## 二、指令传递机制

### 2.1 层级定义
- **鸟 0**: 老大鸟
- **鸟 1**: 直接跟随老大的小鸟
- **鸟 n**: 跟随鸟 n-1 的小鸟

### 2.2 向量更新公式

```
V[n][x] = normalize( V[n-1][x-1] + V[n][x-1] ) / 2
```

### 2.3 Tick 传播机制

| Tick | 事件 |
|------|------|
| **0** | 所有鸟记录当前运动向量 V[n][0] |
| **x** | 鸟n接收鸟n-1在tick x-1的向量，计算 V[n][x] = avg(V[n-1][x-1], V[n][x-1]) |

---

## 三、小鸟行为逻辑

### 3.1 基础行为
- **Reynolds 分离规则**: 避免小鸟之间发生碰撞

### 3.2 Unity 实现要点
- 使用 **Slerp** 进行旋转插值，减少延迟带来的僵硬感

---

## 四、层级分配算法

### 4.1 约束条件
- **最大子节点数**: 3
- **最大深度差**: `ceil(sqrt(总鸟数))`
- **无环**: 使用并查集检测

### 4.2 算法思路

```
类比：平衡 DFS 树 + 负载均衡

1. 老大鸟为根节点
2. 每只小鸟随机选择上级，但需满足：
   - 不形成环（并查集检测）
   - 上级的子节点数 < 3
   - 新增后树的深度差不超过 ceil(sqrt(N))
3. 若随机选择失败，回溯或重新分配
```

## 五、默认参数（可调）

| 参数 | 默认值 | 说明 |
|------|--------|------|
| Tick 频率 | 0.1 秒 (10Hz) | 指令传递的时间间隔 |
| 分离距离 | 2.0 米 | 小鸟之间触发分离的最小距离 |
| 分离力 | 1.0 | 分离力的强度系数 |
| 最大子节点数 | 3 | 每只鸟最多带领的下级数量 |
| 最大深度差 | ceil(sqrt(N)) | N 为总鸟数 |
